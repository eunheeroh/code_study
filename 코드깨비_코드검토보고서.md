## 2. 발견 및 수정한 버그 (8건)

### 🔴 Bug 1: dashboard.js — totalTopics 불일치
- **문제**: 퀴즈 주제가 10개로 늘었는데 `totalTopics = 5`로 하드코딩
- **영향**: 대시보드 "완료 주제" 카드가 `X/5`로 표시 (실제 10개)
- **수정**: `totalTopics = 10`으로 변경

### 🔴 Bug 2: dashboard.js — topicMapping 누락
- **문제**: 새로 추가된 5개 퀴즈 주제(`html-semantic`, `html-form`, `py-variable`, `py-operator`, `py-io`)가 topicMapping에 없음
- **영향**: 해당 주제 퀴즈 결과가 이해도 프로그레스 바에 반영 안 됨
- **수정**: 5개 주제 매핑 추가

### 🔴 Bug 3: dashboard.js — badge-all-topics 조건
- **문제**: 마스터 배지 달성 조건이 `>= 5`인데 주제가 10개
- **영향**: 절반만 완료해도 마스터 배지 해금
- **수정**: `>= 10`으로 변경

### 🟡 Bug 4: tutor.js — layout/form 주제 데이터 누락
- **문제**: HTML 토픽 바에 "레이아웃", "폼 & 입력" 버튼이 있지만, TUTOR_DATA에 해당 키(`layout`, `form`)가 없음
- **영향**: 해당 주제 선택 시 인사 메시지 없이 빈 화면, 질문해도 "준비 중" 메시지만 표시
- **수정**: `layout`(Flexbox 비유 응답)과 `form`(input 비유 응답) 데이터 추가

### 🟡 Bug 5: HTML — 대시보드 초기값 하드코딩
- **문제**: 통계 카드에 `12h`, `78점`, `5일`, `8/20` 등 더미 데이터가 하드코딩
- **영향**: 첫 접속 시 실제 데이터(0)와 불일치 → JS가 갱신하기 전 잠깐 잘못된 숫자 보임
- **수정**: 초기값을 `0h`, `-`, `0일`, `0/10`으로 변경

### 🟡 Bug 6: HTML — 이해도 프로그레스 바 초기값
- **문제**: 90%, 65%, 40%, 55% 더미 데이터가 하드코딩
- **영향**: Bug 5와 동일
- **수정**: 모든 이해도 0%로 초기화, JS에서 실데이터로 갱신

### 🟢 Bug 7: HTML — "JavaScript 기초" 라벨
- **문제**: 파이썬 퀴즈 추가 후 JS 이해도 바가 "JavaScript 기초"로만 표시
- **수정**: "JavaScript / Python"으로 라벨 변경

### 🟢 Bug 8: CSS — 퀴즈 카드 그리드 minmax
- **문제**: 10개 카드가 되면서 `minmax(200px)` 기준으로 2열에 5줄로 길어짐
- **수정**: `minmax(180px)`로 줄여 3열 표시 가능하도록 조정
